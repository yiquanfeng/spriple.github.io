第一步–在原系统中使用tar命令对整个系统进行备份
#

这个过程其实比较容易，只需要注意把一些在新系统中不需要的目录忽略掉即可，并且要保留文件的权限之类的东西 还有要知道tar命令有哪些坑点

sudo tar --exclude=/proc/* \
--exclude=/sys/* \
--exclude=/dev/* \
--exclude=/mnt/* \
--exclude=/run/* \
--exclude=/tmp/* \
--exclude=/lost+found/* \
--exclude=/media/* \
--exclude=/var/cache/pacman/pkg/* \
-czpvf bkp.tar.gz /

–exclude是忽略一些目录 -c 是创造一个tar文件 -z 是选择使用gzip压缩 -p 是保留文件权限 -v 是详细输出 -f 是选择文件的名字 bkp.tar.gz 是备份的文件名 / 是要备份的目录 之后chmod +777 这个脚本然后执行即可

第二步–在新的设备中释放这个压缩包
#

首先进入liveCD中，然后挂载u盘，并且把自己电脑的磁盘也分好区然后mkfs成原设备同样的分区，容量应该可以不一样，因为之后要重新genfstab。 然后把自己电脑的磁盘也挂载好，之后把U盘里面的内容cp过去然后执行

tar -zxvf bkp.tar.gz

即可解压这个文件，之后删除这个文件即可。

第三步，重新制作grub和fstab
#

上一步之后，在liveCD里面执行

genfstab -U /mnt >> /mnt/etc/fstab

[!info] 这里要注意，因为原系统的fstab的内容还在，所以要手动删除一下属于原系统的文件内容

之后把根分区和boot分区都挂载好之后，arch-chroot进去，然后执行

grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

然后exit，reboot，并且拔掉U盘，即可进入新系统
